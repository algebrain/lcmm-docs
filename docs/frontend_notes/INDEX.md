# FRONTEND_NOTES

Этот каталог собирает документацию для фронтенда, который интегрируется с
бекендом, работающим по принципам LCMM.

Главная цель набора документов:

1. зафиксировать инварианты взаимодействия, важные для фронтенда;
2. объяснить ожидаемое поведение бекенда в терминах, полезных для UI;
3. показать, какие клиентские стратегии хорошо согласуются с этими
   инвариантами, не навязывая одну обязательную архитектуру фронтенда.

Во всех документах ниже полезно различать четыре уровня утверждений:

1. инварианты со стороны бекенда;
2. безопасные ожидания со стороны клиента;
3. практичные клиентские стратегии, которые часто удобны;
4. необязательные UI-приемы, которые могут быть уместны, а могут и не быть.

Этот набор не заменяет backend reference docs.
Он извлекает из них только то, что важно для frontend-интеграции.

## Documents

### [QUICKSTART_FOR_FRONTEND](./QUICKSTART_FOR_FRONTEND.md)

Короткая рабочая памятка для frontend-интеграции: что backend действительно
гарантирует, чего не стоит ожидать и с какой минимальной клиентской модели
можно начать.

### [CONTRACT_INVARIANTS](./CONTRACT_INVARIANTS.md)

Что фронтенд может считать устойчивым контрактом взаимодействия с
LCMM-бекендом.

### [HTTP_AND_ERRORS](./HTTP_AND_ERRORS.md)

HTTP-семантика, единый контракт ошибок, повторы запросов и ожидаемое поведение
по статусам.

### [TRACING_AND_DEBUGGING](./TRACING_AND_DEBUGGING.md)

`correlation-id`, `request-id`, трассировка и поддержка сценариев отладки и
обращения в поддержку.

### [REALTIME_AND_SYNC](./REALTIME_AND_SYNC.md)

Какие realtime-инварианты дает бекенд и какие клиентские стратегии из этого
удобно следуют.

### [SECURITY_AND_AUTH](./SECURITY_AND_AUTH.md)

Последствия для фронтенда от аутентификации, защиты от перебора, ограничения
частоты и поведения guard.

### [UI_PATTERNS](./UI_PATTERNS.md)

Практические UI-паттерны, которые хорошо согласуются с инвариантами LCMM.
