# `booking-full/app`

Корень композиции для текущего проекта.

## Что делает приложение

1. Загружает конфиг через `lcmm-configure`.
2. Подготавливает startup state в режимах `--reset` и `--continue`.
3. Собирает `event-bus`, `router`, `read-provider registry`, guard и наблюдаемость.
4. Инициализирует подключенные модули.
5. Поднимает HTTP-маршруты, доступные через `GET`.

## Режимы запуска

```bash
clj -M:run-main -- --reset
clj -M:run-main -- --continue
```

## Стартовое состояние в режиме `--reset`

После запуска в режиме `--reset` приложение приходит к одному и тому же начальному состоянию:

1. существуют пользователи `u-admin` и `u-alice`;
2. существуют слоты `slot-09-00`, `slot-10-00`, `slot-11-00`;
3. списки бронирований, уведомлений и аудита пусты.
